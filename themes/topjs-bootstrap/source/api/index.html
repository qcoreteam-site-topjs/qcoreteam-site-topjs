(static) callback(callback, scope, args, defer, caller, defaultScope) → {Mixed|undefined}

Source:
  • kernel/Util.js, line 45

在指定的作用域上面执行传入的回调函数，当传入的是一个纯函数的时候，让该函数在指定的作用域下 执行，如果传入的函数是字符串，那么我们在传入的作用域对象下寻找是否有这个方法，有就执行。 如果什么都不传，那么我们忽略这个调用。

//下面几个调用时等价的

let fn = this.fn;

TopJs.callback(fn, this, [arg1, arg2]);
TopJs.callback("fn", this, [arg1, arg2]);
Parameters:
Name Type Description
callback function | String

一个函数引用或者一个作用域下的方法名称
scope Object

第一个参数指定的callback的执行作用域，如果第一个参数为字符串，那么必须在这个作用域下 存在名字为callback所指字符串的方法,如果scope为null那么callback将在defaultScope指定的作用域下执行
args Array

传给callback的参数
defer Number

callback延迟调用的毫秒数
caller Object

如果没有显示的提供scope，那么callback调用时将用caller参数进行解析方法
defaultScope Object

默认的作用域，最后的容错对象
Returns:

返回callback的返回值，如果指定defer或者callback不是一个函数则返回undefined

(static) override(target, overrides) → {Object}

Source:
  • kernel/TopJs.js, line 396

用给定的值覆盖目标对象的指定的字段

如果目标的target的是通过TopJs.define定义的类，那么 对象的override方法将会调用 (参考 TopJs.Base#override) 把overrides 当做参数传入

如果传入的target是函数，那么我们默认其为构造函数，overrides将使用TopJs.apply 添加到其的原型对象上

如果目标的target的是通过TopJs.define定义的类的实例化的对象，那么 overrides将复制到实例对象上，这种情况下，方法拷贝会被特使处理，让其能使用TopJs.Base#callParent。

let controller = new TopJs.Controller({ ... });

TopJs.override(controller, {
   forwardRequest: function()
   {
      //一些语句
      this.callParent();
   }
});

如果target不是以上几种，那么overrides将使用TopJs#apply复制到target上

详情请参考 TopJs#define和TopJs.Base#override
